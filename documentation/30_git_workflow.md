# 30. Git Workflow

## 👶 For Freshers (Simple English)
Git is like a time machine for your code. It keeps track of every change. We use branch names like `feature/ai-integration` to work on features separately.

---

## 🧑•💻 For Intermediate Developers (Technical Explanation)
Standard Git conventions:
- `main`: Production-ready code.
- `develop`: Integration branch for developers.
- `feature/*`: Isolation branches for feature work.

---

## 👴 For Senior Developers & Interview Prep (System Design & Code Quality)
Enforce code quality using Git hooks:
- **Pre-commit hooks**: Run linters and tests before allowing commits.
- **Pull Requests (PRs)**: Require review approval and successful CI build status before merging to target branches.\n
